# Phase 4 开发计划 — 实盘对接 + 风控监控

> 目标：从模拟盘到实盘，加上风控体系保障资金安全，系统稳定运行。

---

## 1. 交易执行模块

### 1.1 模拟盘 → 实盘分阶段

| 阶段 | 说明 | 时间 | 资金 |
|------|------|------|------|
| **模拟盘验证** | 纯虚拟交易，验证信号→下单全链路 | 2-4周 | 0 |
| **小资金实盘** | 真实下单，小额验证 | 1-2月 | 1-5万 |
| **正式实盘** | 策略稳定后逐步加仓 | 持续 | 按策略容量 |

### 1.2 券商接口对接

| 方案 | 券商 | 掯P接口 | 成本 | 优先级 |
|------|------|---------|------|--------|
| **QMT（迅投）** | 华鑫/国金等 | Python SDK | 需开户+门槛 | ★★★★★ |
| **easytrader** | 同花顺/通达信 | 模拟操作 | 免费 | ★★★★ |
| **XtQuant** | 华鑫 | Python SDK | 需开户 | ★★★★ |
| **自建接口** | 任意 | 券商HTTP API | 开发成本高 | ★★ |

> 优先对接 QMT，主流且稳定。easytrader 作为低成本备选。

### 1.3 订单管理

```
信号生成 → 仓位计算 → 订单构建 → 下单执行 → 状态跟踪 → 成交确认
```

| 功能 | 说明 |
|------|------|
| **仓位计算** | 按风险预算分配单票仓位（Kelly/固定比例） |
| **订单类型** | 限价单为主，支持市价单 |
| **批量下单** | 月度调仓日一次性下单 |
| **状态跟踪** | 待报/已报/部分成交/全成/撤单 |
| **异常处理** | 超时重报、部分成交补单、撤单重下 |
| **交易日志** | 每笔订单完整记录 |

## 2. 风控体系

### 2.1 风控层级

| 层级 | 规则 | 参数 | 说明 |
|------|------|------|------|
| **个股风控** | 单票最大仓位 | ≤10% | 防集中度 |
| | 单票最大亏损 | -5%止损 | 硬止损 |
| | 单票停牌处理 | 自动剔除 | 阴跌/停牌 |
| **组合风控** | 组合最大仓位 | ≤80% | 留现金缓冲 |
| | 行业集中度 | ≤30% | 行业分散 |
| | 最大回撤阈值 | -10%减仓 | 组合级止损 |
| | 日度波动上限 | 3% | 异常波动预警 |
| **系统风控** | 信号异常检测 | IC<0时暂停 | 信号失效保护 |
| | 数据完整性 | 缺数据不交易 | 防脏数据 |
| | 下单异常 | 连续失败暂停 | 防技术故障 |

### 2.2 风控执行流程

```
组合构建 → 风控检查（仓位/集中度/止损） → 通过/拦截 → 下单 → 实时监控 → 触发止损→执行
```

- **事前**：信号→组合构建时，风控拦截不合规仓位
- **事中**：下单前二次检查，异常暂停
- **事后**：持仓实时监控，触发止损自动执行

## 3. 监控与报警系统

### 3.1 监控指标

| 类别 | 指标 | 频率 | 报警条件 |
|------|------|------|----------|
| **策略表现** | 日度收益 | 每日 | 单日亏损>2% |
| | 累计回撤 | 每日 | 回撤>8% |
| | 持仓IC | 每周 | IC连续2周<0 |
| **风控指标** | 单票仓位偏离 | 每日 | 偏离目标>3% |
| | 行业集中度 | 每日 | >30% |
| | 止损触发次数 | 实时 | 单日>3次 |
| **系统健康** | 数据采集成功率 | 每日 | <90% |
| | 模型预测完成 | 每日 | 未按时完成 |
| | 订单执行成功率 | 实时 | <95% |

### 3.2 报警渠道

| 渠道 | 说明 | 优先级 |
|------|------|--------|
| **企业微信/钉钉** | 实时推送报警消息 | ★★★★★ |
| **邮件** | 每日策略报告 + 异常摘要 | ★★★★ |
| **Dashboard告警面板** | 红色标注异常指标 | ★★★★ |
| **短信** | 紧急情况（大亏损/系统崩溃） | ★★★ |

## 4. 运维体系

### 4.1 自动化任务

| 任务 | 触发方式 | 说明 |
|------|----------|------|
| 数据采集 | cron每日18:00 | 日线+财务增量 |
| 因子计算 | cron每日19:00 | 日度因子更新 |
| ML预测 | cron每月末 | 月度信号生成 |
| 组合构建 | cron每月末 | 调仓日 |
| 下单执行 | cron调仓日开盘前 | 执行交易 |
| 风控检查 | cron每日收盘后 | 组合监控 |
| 报告生成 | cron每周五 | 周度策略报告 |

### 4.2 Docker部署扩展

```yaml
services:
  quant-app:       # 主服务（策略+回测+Dashboard）
  collector:       # 数据采集（定时任务）
  ml-trainer:      # 模型训练（月末触发）
  executor:        # 交易执行（调仓日触发）
  monitor:         # 监控报警服务
  alert:           # 报警推送（企业微信/邮件）
```

### 4.3 数据备份

| 数据 | 备份方式 | 频率 | 保留期 |
|------|----------|------|--------|
| DuckDB文件 | 本机备份 | 每日 | 30天 |
| 交易记录 | 备份+云端 | 每日 | 永久 |
| 模型文件 | 版本管理 | 每次训练 | 全版本 |
| 日志 | 归档压缩 | 每日 | 90天 |

## 6. 开发顺序

| 步骤 | 内容 | 预估时间 | 产出 |
|------|------|----------|------|
| Step 1 | 模拟盘引擎（纯虚拟交易，验证全链路） | 3天 | 信号→模拟下单→记录 |
| Step 2 | 仓位计算 + 组合构建逻辑 | 2天 | 科学的仓位分配 |
| Step 3 | 风控规则定义 + 事前检查 | 3天 | 风控拦截生效 |
| Step 4 | 实时监控 + 止损执行 | 3天 | 事后风控生效 |
| Step 5 | 报警推送（企业微信/邮件） | 2天 | 异常能收到通知 |
| Step 6 | 定时任务编排 + 系统健康检查 | 2天 | 自动化运行 |
| Step 7 | QMT/easytrader接口对接 | 4天 | 真实下单能力 |
| Step 8 | 数据备份 + 日志归档 + 模型版本 | 2天 | 运维保障 |

7. ✅ Dashboard展示交易+风控+监控三看板
