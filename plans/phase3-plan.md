# Phase 3 开发计划 — ML增强 + 因子合成

> 目标：用机器学习模型做因子合成，替代传统等权/IC加权，提升选股alpha。同时加入舆情数据作为辅助信号。

---

## 1. ML因子合成方案

### 1.1 模型选型

| 模型 | 适用场景 | 优势 | 优先级 |
|------|----------|------|--------|
| **LightGBM** | 因子合成主力 | 快、稳、特征交互好 | ★★★★★ |
| **XGBoost** | 对比验证 | 经典GBDT | ★★★★ |
| **线性回归（Ridge/Lasso）** | 基线对比 | 简单可解释 | ★★★ |
| **MLP神经网络** | 进阶尝试 | 非线性拟合 | ★★ |
| **Transformer时序模型** | 实验性 | 捕捉时序依赖 | ★（后期） |

> 主力用 LightGBM，其他作为对比实验。

### 1.2 训练流程

```
特征工程 → 数据清洗 → 滚动训练 → 模型预测 → 信号生成
```

**滚动训练策略**：
- 训练窗口：滚动24个月
- 验证窗口：6个月
- 预测窗口：1个月（每月更新）
- 防止未来信息泄露，严格时间切分

**标签定义**：
- 方案A：未来1个月收益率（回归）
- 方案B：未来1个月收益率排名分组（分类：top 30% / bottom 30%）
- 方案C：未来1个月是否跑赢基准（二分类）
- 默认用方案A（回归），后续对比方案B

### 1.3 特征工程

| 处理步骤 | 说明 |
|----------|------|
| **缺失值填充** | 行业均值填充，极端情况用0 |
| **去极值** | MAD法，3倍MAD外标记为极值，winsorize |
| **标准化** | 截面Z-score标准化 |
| **因子交叉** | 因子间的组合特征（如EP×MOM） |
| **滞后特征** | 因子1月/3月/6月滞后值（捕捉趋势） |
| **行业dummy** | 行业虚拟变量（供模型学习行业效应） |

## 2. 舆情信号模块（可选）

### 2.1 数据来源

| 来源 | 数据 | 成本 | 优先级 |
|------|------|------|--------|
| **东方财富股吧** |帖子情感倾向 | 免费 | ★★★ |
| **新浪财经新闻** | 新闻标题情感 | 免费 | ★★★ |
| **雪球讨论** | 讨论热度+情感 | 免费 | ★★ |
| **爬虫自建** | 定制化数据 | 开发成本 | ★ |

### 2.2 处理流程

```
爬取/采集 → 文本清洗 → 情感分类 → 舆情因子生成 → 入库
```

- 情感分类：先用关键词规则（快速），后期接入LLM API（精准）
- 输出因子：舆情情感得分、舆情热度、情感变化率

> Phase 3 初期重点在ML因子合成，舆情作为可选模块，不影响主线。

## 3. 模型评估与对比

### 3.1 评估维度

| 维度 | 指标 | 说明 |
|------|------|------|
| **预测力** | Rank IC / ICIR | ML信号与未来收益相关性 |
| **选股效果** | 分层回测多空收益 | 实际可用性 |
| **稳定性** | 月度IC波动、IC衰减 | 信号持续性 |
| **过拟合检测** | 训练集vs验证集IC差 | 泛化能力 |
| **对比基线** | vs 等权/IC加权 | ML是否超越传统方法 |

### 3.2 对比实验设计

| 策略 | 因子合成方式 | 说明 |
|------|------------|------|
| **基线1** | 等权合成 | 所有有效因子等权 |
| **基线2** | IC加权 | 按历史IC加权 |
| **基线3** | ICIR加权 | 按ICIR加权 |
| **ML-1** | LightGBM回归 | 单模型 |
| **ML-2** | LightGBM分类 | top/bottom分类 |
| **ML-3** | LightGBM + XGBoost ensemble | 多模型融合 |
