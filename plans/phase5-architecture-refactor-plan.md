## 🎯 改造目标

### 现状分析
- ✅ 当前系统: Dash + Python 一体化架构
- ✅ 已完成功能: 数据采集、因子系统、ML模型、策略回测、实盘监控
- ❌ 问题: 前后端耦合、扩展性差、UI定制能力有限

### 改造目标
1. **架构解耦**: 前端 React + 后端 FastAPI，完全分离
2. **体验升级**: Ant Design Pro 企业级 UI，交互更流畅
3. **性能优化**: 按需加载，API 缓存，前端渲染
4. **扩展增强**: 标准化 API，支持移动端/第三方集成
5. **开发效率**: 前后端并行开发，独立部署

### 技术栈选型

#### 前端技术栈
| 技术 | 选型 | 说明 |
|------|------|------|
| 框架 | React 18 + TypeScript | 类型安全，生态成熟 |
| UI框架 | Ant Design 5.x | 企业级设计系统 |
| 路由 | React Router v6 | 官方推荐 |
| 状态管理 | Zustand | 轻量高效，替代 Redux |
| 数据请求 | Axios + React Query | 缓存 + 自动重试 |
| 图表 | ECharts 5.x | 性能好，中文文档全 |
| 构建工具 | Vite | 比 Webpack 快 10 倍 |
| 代码规范 | ESLint + Prettier | 统一代码风格 |

#### 后端技术栈
| 技术 | 选型 | 说明 |
|------|------|------|
| 框架 | FastAPI | 高性能，自动生成 API 文档 |
| 数据库 | SQLite (当前) / PostgreSQL (扩展) | 当前保持 SQLite，支持后续升级 |
| 缓存 | Redis | API 缓存 + 会话管理 |
| 异步任务 | Celery + Redis | 回测/训练等耗时任务 |
| 认证 | JWT Token | 无状态认证 |
| 文档 | Swagger UI | 自动生成交互式文档 |
| ORM | SQLAlchemy 2.0 | 类型安全的 ORM |

## 🔌 API 接口设计概览

### RESTful API 规范

- 版本前缀: `/api/v1`
- 统一响应格式:
```json
{
  "code": 0,           // 0=成功, 非0=错误码
  "message": "success",
  "data": {},          // 响应数据
  "timestamp": 1234567890
}
```

#### 数据模块 `/api/v1/data`
| 方法 | 路径 | 说明 |

| GET | `/stocks/{code}/quote` | 日线行情 |

## 📦 部署方案

### 开发环境
```bash
# 后端启动 (http://localhost:8000)
cd backend && pip install -r requirements.txt
uvicorn app.main:app --reload

# 前端启动 (http://localhost:3000)
cd frontend && npm install && npm run dev
```

### 生产环境 - Docker Compose
```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    depends_on:
      - redis

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  redis:
    image: redis:alpine
    ports:
      - "6379:6379"

  celery_worker:
    build: ./backend
    command: celery -A app.core.celery worker
    depends_on:
      - redis
```

## ⚠️ 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| SQLite 并发性能 | 中 | 中 | 读多写少场景够用，预留 PostgreSQL 升级方案 |
| 前端开发工作量大 | 高 | 低 | 使用 Pro 组件，优先实现核心功能 |
| 异步任务复杂度高 | 中 | 中 | Celery + Redis 成熟方案，有现成封装 |
| 数据迁移风险 | 中 | 低 | 先只读接入，双写验证后切换 |
